# Project instructions

- After completing requested code changes, run the relevant build and verification checks.
- When the build completes successfully, commit only the changes made for the current request and push the current branch to its configured remote automatically.
- Use a concise, descriptive commit message that states what changed.
- Never include unrelated pre-existing or user-authored working-tree changes in the commit.
- If the build fails, the branch has no configured upstream, authentication fails, or pushing would require a force push, do not push; report the blocker instead.
